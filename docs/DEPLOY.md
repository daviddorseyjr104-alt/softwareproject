# Deploy & Go-Live Runbook (for the client)

This gets the Candidate Finder running at a public HTTPS URL so a GoHighLevel/Flowsa form can reach
it, and lets the client manage everything from a browser. ~20 minutes.

The recommended host is **Render** (simple, HTTPS + secrets + keeps the app alive). Any Node ≥18 or
Docker host works too.

---

## Part A — Deploy on Render (recommended)

1. **Get the code into a Git repo** (GitHub/GitLab). From the project folder:
   ```bash
   git init && git add . && git commit -m "Candidate Finder"
   ```
   Push it to a new private GitHub repo.

2. **Create the service.** Render dashboard → **New +** → **Blueprint** → connect the repo. Render
   reads [`render.yaml`](../render.yaml) and provisions a web service with a persistent disk.

3. **Set the three secrets** (Render → your service → **Environment**):
   | Key | What to enter |
   |---|---|
   | `ADMIN_PASSWORD` | **You choose this** — the password for the settings page. Make it strong. |
   | `SECRET_KEY` | Auto-generated by Render (leave it). Signs sessions + encrypts stored keys. |
   | `WEBHOOK_SECRET` | Auto-generated. This is the secret the form must send. Copy its value. |

   > **Free plan?** Render disks need a paid (Starter) instance. To stay free: delete the `disk:`
   > block in `render.yaml` and set the API keys as env vars here instead
   > (`APOLLO_API_KEY`, `SALESQL_API_KEY`, `INSTANTLY_API_KEY`, `ANTHROPIC_API_KEY`) — the app uses
   > them automatically. Trade-off: keys/jobs entered *in the UI* reset on redeploy without a disk.

4. **Deploy.** Wait for it to go live. Your URL looks like `https://candidate-finder-xxxx.onrender.com`.
   Check `https://<your-url>/health` → should return `{"status":"ok"}`.

---

## Part B — The client sets it up in the browser

1. Go to **`https://<your-url>/admin`** → log in with `ADMIN_PASSWORD`.
2. **API keys** section → paste the four keys (guide + costs in
   [GETTING-KEYS.md](GETTING-KEYS.md)) → **Save keys** → click **Test keys**. Every required
   service should show ✅. (Anthropic is optional — for the AI fit score.)
3. **Companies & roles** section → edit the JSON for the client's real hiring companies + open roles
   → **Save pool** (it's validated before saving).
4. **Behaviour & scoring** → leave **Dry run = true** for now (safe). Tune the score threshold /
   weights if desired.

---

## Part C — Point the form at it

In GoHighLevel/Flowsa, set the Candidate Finder form's webhook/automation to **POST** to:

```
https://<your-url>/webhook/candidate-finder
```

Add a custom header `x-webhook-secret` with the `WEBHOOK_SECRET` value from Part A. Both
GoHighLevel and Flowsa support custom webhook headers.

> A `?secret=` query value is **not** accepted — a secret in a URL leaks into access logs, proxy
> logs, browser history, and the `Referer` header.

---

## Part D — Prove it, then go live

1. Submit the form once with company name **"Test Company"** (this auto-forces dry-run — no email
   sent). In `/admin` → **Recent runs**, confirm the job appears and finishes.
2. Or run the multi-company matcher:
   `curl -X POST -H "x-webhook-secret: YOUR_WEBHOOK_SECRET" https://<your-url>/run-pool`
   and watch it in **Recent runs**.
3. When you're happy with the candidates/scores: in `/admin` set **Dry run = false** and Save.
   Real campaigns will now be created in Instantly on each run.

---

## Health, safety, and limits
- `GET /health` — for uptime monitoring (no auth).
- In production the app **refuses to start** without `WEBHOOK_SECRET`, `ADMIN_PASSWORD`, and
  `SECRET_KEY` — so it can never be accidentally left open.
- API keys are stored **encrypted at rest**; only `SECRET_KEY` (in Render's secret store) can decrypt.
- **Single client per deployment.** To onboard another client, deploy a second copy.
- If the app restarts mid-run, that one in-flight submission is marked `interrupted` (visible in
  Recent runs) rather than silently lost — re-submit it.

## Docker (any host)
```bash
docker build -t candidate-finder .
docker run -p 3000:3000 \
  -e NODE_ENV=production -e WEBHOOK_SECRET=... -e ADMIN_PASSWORD=... -e SECRET_KEY=... \
  -v /host/path/data:/var/data \
  candidate-finder
```
Put HTTPS in front of it (a reverse proxy such as Caddy/nginx, or your platform's TLS).
